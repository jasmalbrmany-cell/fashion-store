## ⚠️ ما يحتاج تنفيذ يدوي منك:

### 🔴 **خطوة 1: إصلاح الأمان (حرج جداً!)**

**المشكلة:** مفاتيح Supabase مكشوفة في `.env.local` على GitHub!

**الحل:**
1. اذهب إلى: https://app.supabase.com/project/jkxfcyngiuefvaxswjxg/settings/api
2. اضغط على **"Reset"** بجانب `anon key`
3. انسخ المفتاح الجديد واحتفظ به

---

### 🟡 **خطوة 2: إصلاح إعدادات Vercel**

**المشكلة:** Vercel يحاول استخدام `pnpm` بدلاً من `npm`

**الحل:**

#### أ) تغيير Build Settings:
1. اذهب إلى: https://vercel.com/jasmalbrmany-cell/fashion-store/settings
2. اضغط على **"General"**
3. ابحث عن **"Build & Development Settings"**
4. غيّر الإعدادات إلى:
   ```
   Framework Preset: Other
   Build Command: npm run build
   Output Directory: dist
   Install Command: npm install
   Development Command: npm run dev
   ```
5. اضغط **"Save"**

#### ب) إضافة Environment Variables:
1. اذهب إلى: https://vercel.com/jasmalbrmany-cell/fashion-store/settings/environment-variables
2. أضف المتغيرات التالية (لكل من Production, Preview, Development):

```
VITE_SUPABASE_URL
القيمة: https://jkxfcyngiuefvaxswjxg.supabase.co

VITE_SUPABASE_ANON_KEY
القيمة: [المفتاح الجديد من الخطوة 1]

VITE_APP_NAME
القيمة: Fashion Hub

VITE_WHATSAPP_DEFAULT
القيمة: 967777123456

FIRECRAWL_API_KEY
القيمة: [مفتاحك من Firecrawl]
```

---

### 🟢 **خطوة 3: إعادة النشر**

1. اذهب إلى: https://vercel.com/jasmalbrmany-cell/fashion-store
2. اضغط على **"Deployments"**
3. اضغط على أحدث deployment
4. اضغط على **"⋯"** (ثلاث نقاط)
5. اختر **"Redeploy"**
6. **مهم:** تأكد من إلغاء تحديد **"Use existing Build Cache"**
7. اضغط **"Redeploy"**
