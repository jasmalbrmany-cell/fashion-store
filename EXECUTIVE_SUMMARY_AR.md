## 🔴 المشكلة الحرجة

### الوصف
```
عند إضافة/تعديل/حذف البيانات في لوحة التحكم:
❌ التغييرات تختفي عند إعادة التحميل
❌ البيانات لا تُحفظ في Supabase
❌ الإعدادات ترجع للقديمة
```

### الجداول المتأثرة
1. cities (المدن)
2. currencies (العملات)
3. categories (الفئات)
4. products (المنتجات)
5. ads (الإعلانات)
6. orders (الطلبات)
7. store_settings (الإعدادات)

### السبب
```sql
-- ❌ السياسات القديمة (خطأ)
CREATE POLICY "Admin Write" ON cities
  USING (auth.role() = 'admin');  -- auth.role() غير موجود!
```

### الحل
```sql
-- ✅ السياسات الجديدة (صحيحة)
CREATE POLICY "Admins can insert cities" ON cities FOR INSERT
  WITH CHECK (
    EXISTS (
      SELECT 1 FROM profiles
      WHERE id = auth.uid() AND role = 'admin'
    )
  );
```

### هل ينشر كل شيء من جديد؟

**لا!** النشر ذكي:

✅ **يُنشر:**
- الكود المعدّل فقط
- Assets الجديدة
- API Routes المعدلة

❌ **لا يُنشر:**
- قاعدة البيانات (منفصلة)
- الصور (في Supabase Storage)
- البيانات المحفوظة

### لماذا البيانات كانت تتغير؟

**قبل الإصلاح:**
```
البيانات في localStorage فقط
   ↓
النشر يمسح localStorage
   ↓
البيانات تختفي ❌
```

**بعد الإصلاح:**
```
البيانات في Supabase
   ↓
النشر لا يؤثر على DB
   ↓
البيانات دائمة ✅
```

## 🔗 الربط بين المتجر وقاعدة البيانات

### كيف يعمل؟

```
1. Environment Variables
   VITE_SUPABASE_URL=https://xxx.supabase.co
   VITE_SUPABASE_ANON_KEY=eyJ...

2. Supabase Client
   const supabase = createClient(url, key)

3. Services Layer
   citiesService.create(city)
      ↓
   supabase.from('cities').insert(...)
      ↓
   RLS Policy يتحقق
      ↓
   ✅ نجح / ❌ فشل
```

### مثال عملي

```typescript
// 1. المستخدم يضيف مدينة
await citiesService.create({
  name: 'صنعاء',
  shippingCost: 3000
});

// 2. Supabase يتلقى الطلب
INSERT INTO cities (name, shipping_cost) 
VALUES ('صنعاء', 3000);

// 3. RLS Policy يتحقق
EXISTS (
  SELECT 1 FROM profiles
  WHERE id = auth.uid() AND role = 'admin'
)

// 4. النتيجة
✅ نجح → البيانات محفوظة
❌ فشل → خطأ 403
```
