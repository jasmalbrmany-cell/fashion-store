# 🏠 كيف تشغل المشروع محلياً

## 📋 المتطلبات:

- Node.js (الإصدار 18 أو أحدث)
- npm (يأتي مع Node.js)
- Git

---

## 🚀 الخطوات:

### 1️⃣ تثبيت Node.js (إذا لم يكن مثبتاً)

```bash
# تحقق إذا كان مثبتاً
node --version
npm --version

# إذا لم يكن مثبتاً، حمّله من:
# https://nodejs.org/
```

---

### 2️⃣ تثبيت المكتبات

```bash
# اذهب لمجلد المشروع
cd extracted_project

# ثبّت المكتبات
npm install
```

**الوقت المتوقع:** 2-3 دقائق

---

### 3️⃣ إعداد ملف البيئة

```bash
# انسخ ملف .env.example
cp .env.example .env.local

# افتح .env.local وعدّل القيم:
nano .env.local
```

**القيم المطلوبة:**
```env
VITE_SUPABASE_URL=https://jkxfcyngiuefvaxswjxg.supabase.co
VITE_SUPABASE_ANON_KEY=[المفتاح الجديد من Supabase]
VITE_APP_NAME=Fashion Hub
VITE_WHATSAPP_DEFAULT=967777123456
```

---

### 4️⃣ تشغيل السيرفر المحلي

```bash
npm run dev
```

**النتيجة:**
```
  VITE v6.4.2  ready in 500 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
  ➜  press h + enter to show help
```

---

### 5️⃣ افتح المتصفح

```
http://localhost:5173/
```

## 🐛 حل المشاكل:

### المشكلة: "npm: command not found"
```bash
# ثبّت Node.js من:
https://nodejs.org/
```

### المشكلة: "Port 5173 is already in use"
```bash
# أوقف السيرفر القديم بـ Ctrl+C
# أو غيّر البورت في vite.config.ts
```

### المشكلة: "Module not found"
```bash
# احذف node_modules وأعد التثبيت
rm -rf node_modules
npm install
```

### المشكلة: "Supabase connection error"
```bash
# تحقق من .env.local
# تأكد أن المفاتيح صحيحة
```

## 🔄 سير العمل المثالي:

### 1. طور محلياً:
```bash
npm run dev
# عدّل الملفات
# المتصفح يتحدث تلقائياً
```

### 2. اختبر البناء:
```bash
npm run build
npm run preview
```

### 3. ارفع على Git:
```bash
git add .
git commit -m "وصف التغييرات"
git push origin main
```

### 4. Vercel ينشر تلقائياً:
```
Vercel يكتشف التغييرات ويبني المشروع تلقائياً
```

## 🎯 نصائح:

1. **استخدم Git Branches** للميزات الجديدة:
   ```bash
   git checkout -b feature/new-feature
   # طور الميزة
   git push origin feature/new-feature
   # افتح Pull Request
   ```

2. **اختبر محلياً قبل الرفع**:
   ```bash
   npm run build  # تأكد أنه يبني بدون أخطاء
   ```

3. **استخدم .env.local** وليس .env:
   ```bash
   # .env.local لا يُرفع على Git
   # .env.example للمثال فقط
   ```

4. **راقب Console** للأخطاء:
   ```
   افتح Developer Tools (F12)
   راقب Console و Network tabs
   ```
